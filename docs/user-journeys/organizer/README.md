# 组织者旅程 (Organizer Journey)

本文档描述了拥有**组织者权限**的用户如何创建、管理和结算活动。组织者同时也拥有普通用户的所有权限。

| 02 | [活动关闭](./02-close.md) | 关闭流程、提案固化、排名计算 |
| 03 | [结算与奖励](./03-settlement.md) | 资金池结算、证书自动发放 |

## 1. 活动创建与发布

**核心目标：** 配置并发布一个合规的活动。
详见：[01-setup.md](./01-setup.md)

- **三步向导**: 基础信息 -> 规则设置 -> 资产设置。
- **状态流转**: Draft -> Pending Review -> Published (资产冻结)。

## 2. 高级活动类型

**核心目标：** 满足复杂的业务需求。
详见：[04-bounty-enterprise.md](./04-bounty-enterprise.md), [05-multi-stage.md](./05-multi-stage.md)

- **悬赏/企业题**: 支持多阶段筛选（海选 -> 决赛）、命题文件下发、自动组队。
- **多赛道**: 并行赛道设置（如：技术赛道 vs 创意赛道）。

## 3. 活动管理与运营

- **仪表盘**: 监控报名与提交数据。
- **规则微调**: 在允许范围内修正非核心规则。
- **公告系统**: 向选手发送通知。

## 4. 结算与闭环

**核心目标：** 确保活动公正结束，奖励准确发放。
详见：[02-close.md](./02-close.md), [03-settlement.md](./03-settlement.md)

- **自动关闭**: 触发 Snapshot 固化提案。
- **自动结算**: 基于 `AssetPool` 和排名自动转账。
- **证书发放**: 系统自动生成并颁发 NFT/PDF 证书。
