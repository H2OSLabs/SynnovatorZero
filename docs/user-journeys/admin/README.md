# 管理员旅程 (Admin Journey)

本文档描述了平台**管理员 (Admin)** 的核心职能，包括内容治理、活动审批、全局配置及系统审计。管理员拥有平台最高权限。

---

## 1. 平台治理

**核心目标：** 维护平台秩序，处理违规内容。
详见：[01-governance.md](./01-governance.md)

- **内容置顶**: 设置首页 Pinned Content。
- **违规处理**: 软删除违规帖子/提案，封禁账号。
- **资源清理**: 清理无主资源。

## 2. 活动审核与监管

**核心目标：** 确保上线活动的合规性与资金安全。

- **准入审核**: 审批 `Pending Review` 的活动，确认 `AssetPool` 冻结。
- **强制干预**: 在紧急情况下强制下架活动或修改规则。

## 3. 系统审计与仲裁

**核心目标：** 处理纠纷，监控系统健康。

- **仲裁服务**: 介入团队纠纷（换队长/解散），处理评分异常。
- **资金审计**: 监控全站资产流动，处理转账失败异常。
- **数据大盘**: 查看平台宏观运营数据（仅管理员可见）。
