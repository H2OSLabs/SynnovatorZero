# 1. 内容浏览

- **角色：** 任何人（含未登录用户）
- **前置条件：** 无

## 1.1 基础浏览与发现

| 用户旅程 | 说明 | 数据操作 |
|---------|------|---------|
| 浏览首页/探索页 | 浏览热门、推荐的**活动**、**提案**和**帖子**。**注意：探索页就是本网站的首页。团队和用户入口不直接出现在探索流中。** | `READ event` / `READ post` (type=proposal/post) |
| 热点追逐 | 点击右侧**“热度榜”**直接跳转至全站最热门内容 | `READ post`（按热度排序，limit 10） |
| 精细搜索 | 使用上方搜索栏进行全局关键词检索 | `SEARCH post/event/user` |
| 内容筛选 | 根据标签、详情属性或自定义内容类型设置过滤器，精准寻找感兴趣的**提案或帖子** | `READ post`（带 tag/type 筛选） |
| 内容排序 | 内容默认按照时间排序，用户也可以点击热度按钮选择按热度排序 | `READ post` (sort_by=created_at/popularity) |
| 查看详情 | 点击内容卡片进入对应内容详情页，可以阅读活动详情页面、参赛提案全文和附件、或用户发布的日常贴 | `READ post` / `READ event` |

### 内容类型定义

在浏览体验中，以下两种内容为**平级关系**，独立展示：

- **帖子 (Post)**: 日常分享、动态更新、讨论贴。
- **提案 (Proposal)**: 具体的项目方案、参赛作品、创意文档。

> 在 UI 上它们是平行的内容分类。

侧边栏入口依次是：**探索**、**营地**、**星球**和**资产**。

| 导航入口 | 说明 | 对应页面/功能 |
|---------|------|-------------|
| 探索 (首页) | 默认首页，聚合推荐与热门内容（不含团队/用户列表） | `ExplorePage` |
| 营地 | 提案和团队集合页，个人工作台 | `CampPage` / `UserDashboard` |

## 1.6 活动状态与可见性规则

> 基于 TC-PERM-020 至 TC-PERM-025 测试用例

### 活动状态展示逻辑

| 活动状态 | 时间条件 | UI展示 | 行为限制 |
|---------|---------|-------|---------|
| **即将开始** | `start_time > now` | “未开始” (Upcoming) | 仅展示，不可报名（除非进入报名期） |
| **报名中** | `enrollment_start <= now <= enrollment_end` | “报名中” (Enrollment Open) | **允许报名**，即便活动尚未正式开始 |
| **进行中** | `start_time <= now < end_time` 且 报名已截止 | “进行中” (Ongoing) | **禁止新报名**，仅限已报名用户提交/更新 |
| **已结束** | `end_time <= now` 或 `status=closed` | “已结束” (Ended) | **禁止报名，禁止修改提案**；展示“活动结束版本”的提案快照 |

### 内容可见性

| 内容类型 | 可见性条件 | 说明 |
|---------|-----------|------|
| Event（活动） | `status=published` | draft 状态的活动对访客不可见 |
| Post（帖子） | `status=published` + `visibility=public` | draft 帖子、private 帖子对非作者不可见 |
| Group（团队） | `visibility=public` | private 团队仅成员可见 |
| Resource（资源） | 继承关联 Post 的可见性 | 若关联帖子不可见，则资源也不可见 |

- **结果：** 用户通过多维导航与发现机制了解平台内容，可决定是否注册参与
